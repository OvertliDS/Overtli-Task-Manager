# File Structure Knowledge Base

## Concurrent route state

| Path                                                  | Responsibility                                                                                           | Invariants                                                                                                                                                                                       |
| ----------------------------------------------------- | -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `src/core/session-scope.mjs`                          | Resolves explicit hook session/thread/conversation ids, environment ids, and stable hashed session keys. | Explicit payload identity wins; environment fallback never selects another session.                                                                                                              |
| `src/core/manager.mjs`                                | Validates workspace/session scope and coordinates guarded lifecycle operations.                          | Legacy adoption is opt-in; terminal parents have terminal internal steps; finalize-then-clear updates durable run/summary state.                                                                 |
| `src/storage/sqlite-store.mjs`                        | Durable concurrent store with WAL mode.                                                                  | Active lookup includes workspace, nullable session id, and active statuses.                                                                                                                      |
| `src/storage/json-store.mjs`                          | JSON fallback store.                                                                                     | Every mutation holds `state.lock`; stale locks older than 30 seconds may be recovered.                                                                                                           |
| `src/core/renderer.mjs`                               | Writes canonical session snapshots and rebuilds the workspace index.                                     | Session snapshots are authoritative; top-level current files are an index.                                                                                                                       |
| `src/hooks/runner.mjs`                                | Resolves hook identity, deduplicates global/workspace invocations, and dispatches lifecycle handlers.    | Hooks without session identity do not touch route state; repeated Stop feedback and Stop errors cannot create an unbounded block loop.                                                           |
| `src/install/agent-block.mjs`                         | Builds and safely patches OTM's marker-delimited root `AGENTS.md` protocol.                              | Preserves all non-OTM text; duplicate or incomplete markers fail closed. The protocol tells Codex to keep one native goal active across the complete OTM route when host controls are available. |
| `AGENTS.md`                                           | Root repository guidance plus OTM's managed execution protocol.                                          | Repository guidance outside OTM markers is maintained manually; only the marker-delimited block is hook-managed.                                                                                 |
| `src/cli/commands.mjs`                                | Exposes install, diagnostics, snapshots, cleanup, and maintenance.                                       | Session-sensitive read/clear commands accept `--session-id`; doctor/snapshot are read-only.                                                                                                      |
| `.codex/overtli-task-manager/sessions/<session-key>/` | Runtime route state and session-owned scratch evidence.                                                  | Generated and ignored by Git; do not edit manually.                                                                                                                                              |

Flow: hook or MCP input -> session resolution -> workspace/session active lookup -> durable store mutation -> canonical session snapshot -> workspace index refresh.
